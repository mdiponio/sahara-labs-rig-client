Pages:

The Pages list contains links to te pages of the rig client administrative interface. The links consist of (briefly - consult the relavant pages for more in-depth descriptions):
  • Status - Shows the current status of the rig client including session detais and exerciser tests statuses.
  • Configuration - Allows the configuration properties of the rig client to be viewed and changed.
  • Logs - Log viewer displaying the latest log messages of the rig client since startup.
  • Diagnostics - Runtime information about the rig client including process details; important paths (class path, library path, ...); memory utilisation; threads and their stack traces; Java virtual machine details and host operating system details.
  • Documentation - All the important information required for effective development, customisation and use of the rig client.

Operations:

The Operations list contains operations which change the temporal state of the rig client. The operations are:
  • Restart - Soft restarts the rig client. This is a soft restart because the Java virtual machine is not restarted. Any in progress sessions are terminated, the scheduling server registration is removed, the server is stopped and all exerciser tests are stopped before all being restarted. The rig type class is instantiated anew causing any all action classes and exerciser tests to be freshly loaded.
    NOTE: Any modifications to the rig client class path, such as dropping a Jar file in './lib', requires the rig client service to be restarted using an operating system specific mechanism.
  • Shutdown - Shuts down the rig client. The rig client service is stopped and to subsequently start the rig client the rig client service must be started in some operating system specific mechanism.
  • Garbage Collection - Peforms a Java virtual machine garbage collection which frees unused memory. The Java virtual machine automically performs this operation and it is safe to manually request this as many times as neccessary.

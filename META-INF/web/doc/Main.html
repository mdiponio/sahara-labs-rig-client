Pages:

The Pages list contains links to pages of the rig client administrative interface. The links consist of:
  • Status - Shows the current status of the rig client including session details and exerciser tests statuses.
  • Configuration - Allows the configuration properties of the rig client to be viewed and changed.
  • Direct Control - Allows the direct control cache (store controller instances) to be viewed and modified. Also allows introspection of controller classes to view and invoke action methods.
  • Logs - Log viewer displaying the latest log messages of the rig client since startup.
  • Diagnostics - Runtime information about the rig client including process details; important paths (class path, library path, ...); memory utilization; threads and their stack traces; Java virtual machine details and host operating system details.
  • Documentation - All the important information required for effective development, customization and use of the rig client.
  • About - Information about the Rig Client.

Operations:

The Operations list contains operations which change the temporal state of the rig client. The operations are:
  • Clear Maintenance - Clears any maintenance states the rig may having putting it back online. A rig client maintenance state is one which requires manual intervention to put it back online. Two maintenance states a rig may be in are:
    • Action failure threshold has been exceeded. If any action has failure more than an allowed limit, the rig is taken offline so it is not subsequently assigned to a user when it may not be in a useable state.
    • The rig is explicitly put it into maintenace.
  • Clean Files - Cleans backup files that the rig client has created. These backup files are:
    • Configuration backups - These files are created in the rig client configuration directories and backup the configuration state before a change is made.
    • Log backups - When using the RolledFile logger, when the log files reachs the configured maximum log file size, the log file is rolled to a backup file (has a number apended to it).
  • Garbage Collection - Performs a Java virtual machine garbage collection which frees unused memory. The Java virtual machine automatically performs this operation and it is safe to manually request this as many times as necessary.
  • Restart - Soft restarts the rig client. This is a soft restart because the Java virtual machine is not restarted. Any in progress sessions are terminated, the scheduling server registration is removed, the server is stopped and all exerciser tests are stopped before all being restarted. The rig type class is instantiated anew causing any all action classes and exerciser tests to be freshly loaded.
    NOTE: Any modifications to the rig client class path, such as dropping a Jar file in './lib', requires the rig client service to be restarted using an operating system specific mechanism.
  • Shutdown - Shuts down the rig client. The rig client service is stopped and to subsequently start the rig client the rig client service must be started in some operating system specific mechanism.
    HINT: On Windows, to start the rig client open 'Control Panel' -> 'Administrative 'Tools' -> 'Services' and in the list of services, start the rig client.
    HINT: On Linux, to start the rig client execute '/etc/init.d/rigclient start' with root privileges.
